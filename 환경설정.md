## Python virtual enviroment

파이썬 가상환경이란, 프로그램을 내 컴퓨터 전체에 설치하는게 아니라, 특정 폴더에서만 실행할 수 있도록 설치하는 것   

- 의존성 관리   
3개의 프로그램 A, B, C가 있다고 치자.   
C는 A 1.0 버전, B 3.32 버전을 설치해야만 실행되는 프로그램이다.   
그런데, 어느날 A가 업데이트를 했다. A는 2.1 버전이 되었다.   

C 프로그램은 작동하지 않는다!

이런 상황을 의존성 문제가 발생했다고 한다.   
이걸 관리하기 위해서 가상환경을 설치한다.

'집만들기' 프로젝트의 가상환경에는 A 1.0버전, B 3.32버전, C 1.1버전을 다운로드 받아둔다.

'집부수기' 프로젝트의 가상환경에는 A 2.1버전, B 3.32버전, C 2.3버전을 다운로드 받는다.

똑같은 이름의 프로그램인데도 다른 버전의 프로그램을 여러 개 설치할 수 있게된다. 이러면, 나중에 무언가 달라지더라도, 이미 가상환경에 설치된 프로그램은 건들이지 않으므로 실행이 안 되거나 하는 문제가 사라진다.

1. Command Prompt에 'python -m venv venv' 입력 후 엔터
2. 기다리면, 왼쪽 Explorer에 venv가 생김
3. Command Prompt에 'cd venv/scripts' 입력 후 엔터
4. Command Prompt에 'activate' 입력 후 엔터

python -m venv {name}   
파이썬 가상환경 만드는 명령어. name은 원하는 이름 설정. 보통 venv 또는 myenv 등 사용함

cd venv/scripts   
Linux 및 Windows 등에서 사용하는 DOS 명령어   
Change Directory의 약자로, 뒤에서 지정한 어떤 폴더로 이동한다

actiavate   
윈도우 cmd에서 .bat 또는 .exe 등으로 끝나는 실행 파일을 실행한다.   
이경우 venv/scripts/ 안에 Activate.bat 이라는 파일이 있음.   
만약 파일 이름이 code.exe 라면 code를 입력하면 실행됨

## requirements.txt

미리 만들어진 프로그램을 다운로드 받아야함.   
git clone 할 때 같이 받은 requirements.txt 파일이 있을 것   
그 파일은 이번 프로젝트를 실행하는데 필요한 프로그램 목록들이다.   
보면 각 버전들도 같이 적혀있음을 알 수 있음   

1. pip install -r requirements.txt
